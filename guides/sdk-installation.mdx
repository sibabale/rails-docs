Rails provides **five** official, auto-generated SDKs. Each is type-safe and exposes the same public API. Pick the one that matches your stack.

## TypeScript

Install the SDK using your preferred package manager:

<Tabs>
  <Tab title="npm">
    ```bash
    npm install @rails/sdk
    ```
  </Tab>
  <Tab title="pnpm">
    ```bash
    pnpm add @rails/sdk
    ```
  </Tab>
  <Tab title="yarn">
    ```bash
    yarn add @rails/sdk
    ```
  </Tab>
  <Tab title="bun">
    ```bash
    bun add @rails/sdk
    ```
  </Tab>
</Tabs>

Initialize the client:

```typescript
import { Rails } from "@rails/sdk";

const rails = new Rails({
  apiKey: process.env.RAILS_API_KEY
});
```

## Go

Add the Rails Go SDK to your module:

```bash
go get github.com/rails/rails-go
```

Initialize the client:

```go
import (
  "os"
  "github.com/rails/rails-go"
)

client := rails.NewClient(rails.WithAPIKey(os.Getenv("RAILS_API_KEY")))
```

## Java

Add the Rails dependency to your `pom.xml` (Maven) or build file:

```xml
<dependency>
  <groupId>com.rails.api</groupId>
  <artifactId>rails</artifactId>
  <version>0.1.0</version>
</dependency>
```

Initialize the client:

```java
import com.rails.api.RailsClient;

RailsClient client = RailsClient.builder()
  .apiKey(System.getenv("RAILS_API_KEY"))
  .build();
```

## Kotlin

Use the same Maven dependency as Java. Initialize the client:

```kotlin
import com.rails.api.RailsClient

val client = RailsClient(apiKey = System.getenv("RAILS_API_KEY")!!)
```

## C#

Add the Rails NuGet package:

```bash
dotnet add package Rails
```

Initialize the client:

```csharp
using Rails;

var client = new RailsClient(Environment.GetEnvironmentVariable("RAILS_API_KEY"));
```

## Configuration (all SDKs)

- **API key** – Required. Set via constructor/options or `RAILS_API_KEY` (where supported).
- **Environment** – Pass **sandbox** or **production** where required (e.g. user creation). **Defaults to sandbox** when not provided, so development and testing are safe by default.
