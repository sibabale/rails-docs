## Welcome to the Rails API

The Rails API is the **public, SDK-supported** surface for building banking applications. It is built for **business owners (startup founders)** who run a bank: you create users and accounts, move money, and list or retrieve data for operations and for showing your own customers (e.g. “your transactions”). Internal, admin, and system endpoints are not exposed via the SDK and are not documented here.

## API Overview

This reference documents only endpoints that are **Public (SDK)**. Using the SDK is the supported way to call the API; it exposes only approved operations and enforces API key and environment requirements.

### Users

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/users` | Create a user (customer/employee). Requires `X-Environment`. |

### Accounts

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/accounts` | Create an account for a user. |
| GET | `/api/v1/accounts` | List accounts (e.g. by `user_id`). |
| GET | `/api/v1/accounts/:id` | Retrieve an account. |
| PATCH | `/api/v1/accounts/:id` | Update account status (e.g. suspend). |
| DELETE | `/api/v1/accounts/:id` | Close an account. |
| POST | `/api/v1/accounts/:id/deposit` | Deposit into account. |
| POST | `/api/v1/accounts/:id/withdraw` | Withdraw from account. |
| POST | `/api/v1/accounts/:id/transfer` | Transfer to another account. |
| GET | `/api/v1/accounts/:account_id/transactions` | List transactions for an account. |

### Transactions

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/transactions` | List transactions by organization (paginated). |
| GET | `/api/v1/transactions/:id` | Retrieve a single transaction. |

## Environment

Use the `X-Environment` header with **sandbox** or **production**. **Environment defaults to sandbox** when not provided, so development and testing are safe by default. Send the header explicitly when you need production.

## Authentication

All public endpoints require **API key** authentication. Send your API key in the `X-API-Key` header. Unauthenticated access is not supported. The recommended way to call the API is with one of the official SDKs (TypeScript, Go, Java, Kotlin, C#), which handle headers and serialization for you.

### API Keys

- **Test keys** – For sandbox (prefix: `rails_test_`)
- **Live keys** – For production (prefix: `rails_live_`)

## Request / Response

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `X-API-Key` | Yes | Your API key. |
| `X-Environment` | When applicable | `sandbox` or `production`. |
| `Content-Type` | For JSON bodies | `application/json`. |

Responses may include `x-correlation-id` for support and debugging.

### Error format

Error responses use standard HTTP status codes and a consistent body:

```json
{
  "status": "error",
  "code": "VALIDATION_FAILED",
  "message": "Something went wrong"
}
```

| Status | Description |
|--------|-------------|
| 200, 201 | Success |
| 400 | Bad Request – Invalid parameters |
| 401 | Unauthorized – Invalid or missing API key |
| 403 | Forbidden – Insufficient permissions |
| 404 | Not Found – Resource doesn't exist |
| 500 | Server Error |

## Official SDKs

Rails provides **five** official, auto-generated SDKs. Use the one that matches your stack:

| Language   | Package / repo     | Install / usage |
|-----------|--------------------|------------------|
| **TypeScript** | `@rails/sdk` (npm) | `npm install @rails/sdk` |
| **Go**         | `github.com/rails/rails-go` | `go get github.com/rails/rails-go` |
| **Java**       | `com.rails.api` (Maven) | Add dependency, then `RailsClient.builder().apiKey(...).build()` |
| **Kotlin**     | `com.rails.api` (Maven) | Same as Java; use `RailsClient(apiKey = ...)` |
| **C#**         | Rails NuGet package | `dotnet add package Rails`, then `new RailsClient(apiKey)` |

Example with the TypeScript SDK:

```typescript
import { Rails } from "@rails/sdk";

const rails = new Rails({ apiKey: process.env.RAILS_API_KEY });

// Create user (include environment for user creation)
const user = await rails.users.create({
  firstName: "Jane",
  lastName: "Doe",
  email: "jane@example.com",
  password: "secure-password",
  environment: "sandbox"
});

// Create account
const account = await rails.accounts.create({
  userId: user.userId,
  accountType: "checking"
});

// List account transactions
const transactions = await rails.transactions.listByAccount(account.id);
```

## OpenAPI spec

The public API is defined in [openapi.json](/api-reference/openapi.json) in this repo. It is kept in sync with the SDK: only Public (SDK) endpoints appear there.
